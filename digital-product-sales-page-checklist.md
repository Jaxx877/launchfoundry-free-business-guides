# Digital Product Sales Page Checklist

## Offer clarity

- [ ] Name the intended buyer and active situation.
- [ ] Define one bounded, visible result.
- [ ] Explain the workflow or mechanism.
- [ ] State what still depends on the buyer, market, or outside conditions.

## Evidence

- [ ] Show real product excerpts or screenshots.
- [ ] Include a current file or module list.
- [ ] Label completed examples clearly.
- [ ] Use testimonials only with real permission and accurate context.
- [ ] Remove unsupported statistics, fake activity, and invented scarcity.

## Buying information

- [ ] Display total price, currency, and billing model.
- [ ] Explain delivery timing and format.
- [ ] State prerequisites, support, and refund or replacement terms.
- [ ] Link privacy, terms, refunds, and support before checkout.

## Journey test

- [ ] Test the page on mobile.
- [ ] Open every link.
- [ ] Verify the correct Stripe product, price, currency, receipt, and post-payment destination.
- [ ] Download from a clean session and follow the start path.
